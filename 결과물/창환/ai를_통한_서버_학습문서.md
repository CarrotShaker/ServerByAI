# 학습 내용 정리


## 1.시놀로지 서버 vs Supabase 비교 ai 질답

### 시놀로지로 API 뽑는 과정

대략적인 흐름은 이렇습니다:

1. **서버 환경 구성** - Docker 설치 (시놀로지는 Docker 지원 좋음)
2. **백엔드 프레임워크 선택** - Node.js(Express/Fastify), Python(FastAPI), Kotlin(Ktor) 등
3. **DB 설치** - PostgreSQL, MySQL 등 Docker로 띄우기
4. **DDNS 설정** - 시놀로지 자체 DDNS 또는 외부 서비스
5. **포트포워딩** - 공유기에서 443, 80 등 열기
6. **SSL 인증서** - Let's Encrypt (시놀로지 내장 지원)
7. **API 개발 및 배포**

---

### 각각 장단점

| 항목 | 시놀로지 | Supabase |
|------|---------|---------|
| **초기 설정** | 복잡 (네트워크, 인증서 등) | 거의 없음 (5분이면 시작) |
| **비용** | 전기세만 | 무료 플랜 있음 (DB 500MB, API 무제한) |
| **API 개발** | 직접 코딩 필요 | REST/GraphQL **자동 생성** |
| **인증** | 직접 구현 | Auth 내장 (소셜로그인 포함) |
| **실시간** | 직접 구현 (WebSocket 등) | Realtime 내장 |
| **MCP 연동** | 별도 구성 필요 | 공식 MCP 있음 |
| **학습 가치** | 높음 (실제 서버 운영 경험) | 낮음 (추상화됨) |
| **커스터마이징** | 완전 자유 | 제한적 |
| **장애 대응** | 직접 해결해야 함 | 알아서 관리됨 |
| **집 인터넷 의존** | O (외부에서 집 서버로 요청) | X |

---

### 토이프로젝트 3~4개 API 기준으로는?

**Supabase 추천** - 이유:
- API 3~4개 수준이면 **테이블 만들면 REST API 자동 생성**됨
- Compose 연습이 목적이라면 서버에 시간 낭비 없이 **클라이언트 개발에 집중** 가능
- MCP 연결하면 AI로 테이블 설계, 쿼리 작성까지 빠르게 처리 가능
- 무료 플랜으로 충분

**시놀로지가 유리한 경우:**
- 백엔드 로직을 직접 짜면서 서버 개발 공부도 같이 하고 싶을 때
- 특수한 비즈니스 로직이 필요할 때 (Supabase Edge Function으로도 가능하긴 함)
- 데이터를 외부 서비스에 올리기 싫을 때

---

### 결론

**Compose 연습이 목적이라면 Supabase가 압도적으로 편합니다.**

시놀로지는 "나중에 서버 개발도 배우고 싶다"는 목표가 생겼을 때 도전해도 늦지 않아요. 지금 당장 API 3~4개 뽑고 앱 만들려면 Supabase가 훨씬 빠릅니다.
